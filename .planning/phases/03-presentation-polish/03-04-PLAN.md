<objective>
Implement D-18 font scaling: client reports fonts are "gigantes" (too big) on real TV hardware despite Phase 2's `clamp()` with `vw` preferred values. Root cause: `vw` is viewport-relative, so on a 1920px viewport a `3vw` preferred becomes ~57px — the card values look huge because each card is only half the viewport width but the text scales with the full viewport.

Fix: wrap each half-panel with `@container` (Tailwind 4 native), replace `vw` with `cqi` (container inline axis) in the preferred-value slot of all `clamp()` expressions inside, and lower the ceilings so the largest value is visibly smaller. This makes text scale relative to each card's actual width (~half viewport) instead of the full viewport, producing the "smaller overall" effect D-18 requires.

Depends on 03-06 (which depends on 03-03): shares both files with 03-03 and 03-06, so this plan runs last in the sequential chain that edits company-display.tsx and performer-presentation.tsx.

Purpose: Proportional reduction of all text in presentation mode, CSS-only.
Output: `@container` wrappers and updated `clamp()` values using `cqi`.
</objective>

<!-- Tailwind 4 container query syntax (verified in research) -->

```tsx
// Parent: add @container class (sets container-type: inline-size automatically)
<div className="@container relative h-full overflow-hidden">
  {/* children resolve cqi against this container's inline (width) axis */}
  <span className="text-[clamp(0.9rem,1.4cqi,1.4rem)]">label</span>
  <p className="text-[clamp(1.2rem,2.8cqi,3rem)]">value</p>
</div>
```

<!-- Current Phase 2 clamp() patterns (to be replaced) -->

- Card label: `text-[clamp(1.5rem,1.3vw,2rem)]` ← current
- Card value: `text-[clamp(1.5rem,3vw,4rem)]` ← current
- City/rank: `text-[clamp(0.75rem,0.9vw,1rem)]` ← current
- Artist name: `text-[clamp(1.5rem,3vw,4rem)]` ← current

<!-- Target values (research recommendation — starting point, implementer may tune) -->

- Card label: `text-[clamp(0.9rem,1.4cqi,1.4rem)]`
- Card value: `text-[clamp(1.2rem,2.8cqi,3rem)]`
- City/rank: `text-[clamp(0.65rem,0.9cqi,0.9rem)]`
- Artist name: `text-[clamp(1rem,2.5cqi,2.5rem)]`

**Pitfall 7 (Tailwind JIT):** Arbitrary values must be complete string literals in the className. Template-string construction like `` `text-[clamp(${x}rem,${y}cqi,${z}rem)]` `` will NOT be scanned. Every clamp() must be a literal.

**Pitfall 2 (cqh requires height):** Use `cqi` (inline / width) not `cqh` (height). Card widths are resolved from the grid, heights may be fractional.

**Where @container goes:** The container-query containment context must be the half-panel wrapper (the div that directly contains each card grid). For company-display.tsx post-03-02, this is the "LEFT HALF" and "RIGHT HALF" inner z-10 divs. For performer-presentation.tsx, this is the left-half wrapper (the metric grid parent).

**GRID-05 readability floor:** Phase 2 requires labels ≥ 24px equivalent at 1080p. With `cqi`, a card ~940px wide × 1.4cqi = ~13px preferred — but the floor of `0.9rem` (14.4px) plus clamp protection means the label will never go below 14.4px and can grow up to 22.4px (1.4rem). This is BELOW the Phase 2 GRID-05 floor of 24px at 1080p. Implementer may need to raise the floor to `1.5rem` (24px) to preserve GRID-05 compliance. **This is a trade-off the client explicitly accepted — D-18 says "smaller overall" even if it temporarily relaxes GRID-05.** Document the trade-off in the implementation comment and proceed with the research values. Client-side tuning is expected on the real TV.
