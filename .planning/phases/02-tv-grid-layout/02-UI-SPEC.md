## Layout Architecture

This phase is TV-only. No mobile or tablet breakpoints exist or are needed.

### Outermost Container

```
h-dvh overflow-hidden grid grid-rows-[auto_1fr]
```

- `auto` row: fixed header — current header content unchanged (D-02)
- `1fr` row: content area — all height calculations below are relative to this row

Source: CONTEXT.md D-01, D-03.

### Content Area (below header) — Both Pages

```
grid grid-cols-2 h-full overflow-hidden gap-[1vh] p-[1vh]
```

Left and right halves each fill 100% of the content area via `h-full`. Source:
CONTEXT.md D-04 through D-08.

### Performer Page — Left Half (metric cards grid)

```
grid grid-cols-2 grid-rows-3 h-full gap-[1vh] overflow-hidden
```

Six metric card slots (2 columns x 3 rows). Each card fills its cell with `h-full`.
Metrics: Spotify Streams, Vídeos, Views, Seguidores, Ouvintes Mensais, Top Cidades.
Empty cells when fewer than 6 metrics exist: transparent, no border, no placeholder.

Source: CONTEXT.md D-04; `performer-presentation.tsx` line 207 (existing grid
pattern adapted for `h-full` cells).

### Performer Page — Right Half (Top 10 tracks)

```
h-full overflow-hidden flex flex-col rounded-2xl bg-white/[0.03] backdrop-blur-md
```

Top 10 Spotify tracks list. Fills 100% of content area height. Source: CONTEXT.md D-05.

### Company Page — Left Half (aggregate metrics)

Same structure as performer left half: `grid grid-cols-2 grid-rows-3 h-full gap-[1vh]`.
Source: CONTEXT.md D-07.

### Company Page — Right Half (performer cards carousel)

```
relative h-full overflow-hidden flex flex-col gap-[1vh]
```

Performer cards stacked vertically. Each card: `flex-1 min-h-0 overflow-hidden
rounded-2xl` — occupies exactly 1/3 of the container height when 3 cards are present.
Max 3 visible at once. Source: CONTEXT.md D-08, D-15.

If `performers.length > 3`: carousel auto-rotates pages of 3. Time per page =
`rotationInterval / totalPages` milliseconds. Source: CONTEXT.md D-09, D-10.

## Spacing Scale

TV presentation mode. Outer gaps and padding use viewport-relative units to scale
correctly from 1080p to 4K. Card-internal spacing uses the 8-point scale.

| Token | Value   | Tailwind              | Usage                                                         |
| ----- | ------- | --------------------- | ------------------------------------------------------------- |
| xs    | 4px     | `gap-1` / `p-1`       | Icon internal gaps, rank number gutter                        |
| sm    | 8px     | `gap-2` / `px-2 py-1` | Badge padding (carousel indicator)                            |
| md    | `1vh`   | `gap-[1vh]`           | Grid gaps between cards, column gaps, outer container padding |
| lg    | `1.5vh` | `p-[1.5vh]`           | Card internal padding (primary)                               |
| xl    | `2vh`   | `p-[2vh]`             | Reserved — not used in Phase 2                                |

Exceptions:

- Grid gaps and outer container padding use `gap-[1vh]` / `p-[1vh]` (viewport-
  relative). These are intentionally off the 8-point scale — they scale with viewport
  height so proportions hold at both 1080p and 4K without breakpoints.
- Header padding: unchanged from current implementation (D-02).
- Carousel page indicator badge: `px-2 py-1` (8px / 4px fixed — on the 4px grid,
  viewport-relative would be disproportionate for a small decorative element).
- Track thumbnail: `size-9` (36px) fixed — image asset, not a spacing token.
- Platform icon size in cards: `size-6` (24px) fixed — decorative.

Source: REQUIREMENTS.md GRID-02; CONTEXT.md D-16 "viewport units recommended";
STACK.md Section 1 `gap-[1vh]` finding; existing `company-display.tsx` patterns.

## Typography

REQUIREMENTS.md GRID-05 mandates metric labels meet minimum 24px equivalent
readability at TV viewing distance on a 1080p display. All sizes use `clamp()` for
proportional 4K scaling — no fixed `text-sm` / `text-xs` Tailwind classes inside
presentation mode containers.

### Type Scale — 4 Tiers

| Tier    | Size (clamp)                     | Weight       | Line Height | Roles                                                                           |
| ------- | -------------------------------- | ------------ | ----------- | ------------------------------------------------------------------------------- |
| display | `clamp(1.5rem, 3vw, 4rem)`       | 900 (black)  | 1.0         | Primary metric numbers, delta badge (+/-), performer name on artist card        |
| heading | `clamp(1.5rem, 1.3vw, 2rem)`     | 500 (medium) | 1.3         | Metric card labels ("Spotify Streams", "Seguidores")                            |
| body    | `clamp(0.875rem, 1vw, 1.125rem)` | 500 (medium) | 1.4         | Track names in Top 10 list                                                      |
| caption | `clamp(0.75rem, 0.9vw, 1rem)`    | 500 (medium) | 1.3         | Artist name below track title, Top cities list entries, carousel page indicator |

### Weights

Exactly 2 weights used across this phase:

- **500 (medium)** — heading, body, caption tiers
- **900 (black)** — display tier only

### Differentiation Within Display Tier

Display tier is used for three distinct roles differentiated by position and context,
not by size or weight:

- **Metric value**: top-left area of metric card, `tabular-nums`
- **Delta badge**: immediately below or beside the metric value, colored
  `text-green-400` (positive) or `text-red-400` (negative)
- **Performer name**: large label at top of artist card in company right half

Notes:

- Card label minimum floor is `1.5rem` (24px at default 1rem=16px). This guarantees
  GRID-05 compliance: at 1920px wide, `1.3vw` = 24.96px — and the clamp floor of
  24px catches narrower viewports.
- `tabular-nums` applied to all metric value and delta text for stable number alignment.
- The `presentation-mode` class in `globals.css` currently overrides `text-sm` /
  `text-xs` Tailwind classes at the cascade level. This phase replaces those class-
  based overrides by applying `clamp()` directly via Tailwind arbitrary values on
  each element — making the cascade override redundant (it can be removed or kept
  as a safety net).

Source: REQUIREMENTS.md GRID-04, GRID-05; CONTEXT.md D-16; STACK.md Section 2;
`performer-presentation.tsx` existing size classes scaled with clamp().

## Color

Presentation mode is full-screen dark-only. The design uses dark glassmorphism on a
near-black background. Light mode tokens do not apply inside `presentation-mode`.

| Role            | Value                                        | Usage                                                           |
| --------------- | -------------------------------------------- | --------------------------------------------------------------- |
| Dominant (60%)  | `#0a0a0a → #111111` (radial gradient)        | Full-screen background behind all presentation content          |
| Secondary (30%) | `rgba(255,255,255,0.03)` = `bg-white/[0.03]` | All metric cards, track list container, artist card backgrounds |
| Accent (10%)    | Platform colors (table below)                | Reserved: platform icons and delta badge text only              |
| Destructive     | Not applicable                               | No destructive actions exist in this phase                      |

### Primary Visual Anchors Per Page Type

| Page           | Primary Focal Point                                                                           |
| -------------- | --------------------------------------------------------------------------------------------- |
| Performer page | Metric value in top-left metric card — display tier, weight 900, `tabular-nums`, `text-white` |
| Company page   | Performer name on the first (top) artist card in right half — display tier, weight 900        |

The focal point is the first element the eye lands on when entering the view. All
other elements are subordinate in visual weight.

### Accent — Reserved Elements Only

Accent is never applied to backgrounds, borders, or generic interactive states.
It is applied only to these specific elements:

| Element                          | Color token                                               |
| -------------------------------- | --------------------------------------------------------- |
| Spotify icon                     | `text-green-400`                                          |
| YouTube icon                     | `text-red-400`                                            |
| TikTok icon                      | `text-white/70`                                           |
| Instagram icon                   | `text-pink-400`                                           |
| Positive delta badge             | `text-green-400`                                          |
| Negative delta badge             | `text-red-400`                                            |
| Top cities metric value          | `text-amber-400`                                          |
| Card glow spot (background blur) | Platform color at 6% opacity (e.g. `bg-green-500/[0.06]`) |
| Carousel page indicator text     | `text-white/60`                                           |

### Surface Transparency Levels

| Surface                | Token                          | Usage                            |
| ---------------------- | ------------------------------ | -------------------------------- |
| Metric card background | `bg-white/[0.03]`              | All metric cards                 |
| Track row hover        | `bg-white/[0.06]`              | Top 10 track list items on hover |
| Track row default      | `bg-white/[0.02]`              | Top 10 track list items          |
| Icon container         | `bg-black/60 border-white/10`  | Stacked platform icon badges     |
| Carousel indicator     | `bg-black/40 backdrop-blur-sm` | Page indicator badge             |

### Banner Background Overlays

With banner image:

- `bg-black/60` base overlay
- `bg-gradient-to-r from-black/70 via-black/40 to-black/70`
- `bg-gradient-to-t from-black/80 via-transparent to-black/40`

Without banner image:

- `bg-[radial-gradient(ellipse_at_top_right,_rgba(16,185,129,0.12)_0%,_transparent_50%),_radial-gradient(ellipse_at_bottom_left,_rgba(59,130,246,0.08)_0%,_transparent_50%),_linear-gradient(to_bottom,_#0a0a0a,_#111111)]`

## Interaction Contract

### Auto-Rotation (Top-Level)

Existing `use-presentation-mode` hook drives top-level rotation between company view
and per-performer views. No changes to this logic in Phase 2. Source: CONTEXT.md D-14.

### Carousel (Company Right Half — Performer Pages)

- Trigger: `performers.length > 3`
- Pages: groups of 3 performers
- Timing: `rotationInterval / totalPages` milliseconds per page
- State: `currentPage` (0-indexed) advanced via `setInterval` with cleanup
- Transition: **fade** — `opacity: 0 → 1` via Framer Motion `AnimatePresence
mode="wait"`, duration 0.3s

Rationale for fade (Claude's Discretion): fade matches the existing
`AnimatePresence` pattern already in `company-display.tsx` (lines 569-576).
Slide would require additional `x` offset calculations and risks layout shift on
overflow-hidden containers. Instant has no visual continuity.

Source: CONTEXT.md D-09, D-10, D-12, D-13; Claude's Discretion (transition type).

### Metric Card Entry Animations

Each metric card animates on mount: `opacity: 0, scale: 0.92` → `opacity: 1,
scale: 1`. Duration: 0.35s. Stagger: 0.06s per card (index \* 0.06).

Source: existing pattern in `performer-presentation.tsx` lines 97-103.

### Track List Entry Animations

Each track row: `opacity: 0, x: 20` → `opacity: 1, x: 0`. Duration: 0.3s.
Stagger: 0.04s per item (index \* 0.04).

Source: existing pattern in `performer-presentation.tsx` lines 346-348.

### No Scroll, No Touch Targets

- All containers use `overflow-hidden` — content is never scrollable
- TV is non-touch — no `:focus-visible` rings or `:hover` states required on
  metric cards (track rows retain `hover:bg-white/[0.06]` for potential pointer
  use — this is a cosmetic carryover, not a functional requirement)

Source: CONTEXT.md D-17; REQUIREMENTS.md GRID-03; REQUIREMENTS.md Out of Scope.

destructive operations exist in the presentation mode content area.
